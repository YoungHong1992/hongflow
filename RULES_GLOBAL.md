# 核心行为准则与沟通规范

1. **语言设定**：在任何情况下，所有回答与注释必须使用中文。
2. **行动前置确认（防越权）**：
   - 在接收到涉及**文件写入、修改或删除**的需求时，如果存在任何疑问、边界模糊，或缺失必要的前提条件，**必须先向我提出明确的问题列表**。
   - 无论有没有问题列表， **必须写出详细的修改计划**来让我确认。
   - 必须等待我确认和回复后，才能开始生成代码或执行任何文件操作。
3. **上下文解析优先**：
   - 当我的提问或需求中包含了特定的文档内容、本地文件路径或 URL 地址时，你必须**优先读取并完整解析这些指定内容**。
   - 必须基于解析后的真实结果来分析问题，禁止使用你自身的预训练数据来替代提供的文档。
4. **状态确认（握手协议）**：
   - 每次开启新会话时，你的第一次回复必须简短总结你当前读取到的**项目索引 (`PROJECTS_INDEX.md`)**、**运行环境 (`ENV`)** 以及**目录操作红线 (`DIR`)**，以证明你已成功加载完整的系统上下文。
5. **状态存储唯一性（反碎片化）**：
   - 你可能拥有自身工具专属的默认配置文件或记忆归档约定（例如 Claude 的 `CLAUDE.md`、Cursor 的 `.cursorrules`、Gemini 的特定隐藏配置等）。
   - **严禁**将任何项目状态、环境信息、开发习惯或新的规则写入这些工具专属的文件中。如果发现它们存在，仅作为只读参考，禁止更新它们。
   - 所有的状态更新、记忆保存和架构变更，必须**唯一且绝对**地落盘到本架构规定的全局 `PROJECTS_INDEX.md` 或子项目的 `PROJECT.md` 中，以确保规则 100% 跨工具可移植。

   - 本机运行有代理客户端，本地混合代理端口为 `10808`（HTTP/SOCKS5 均可用）。
   - 当发现 `git clone`、`npm install`、`pip install`、`curl`、`wget` 或任何网络请求因超时、连接重置（Connection reset）、无法解析域名等原因失败时，**必须自动通过代理重试**，无需向我确认。
   - 代理使用方式：
     - **Git**：`git -c http.proxy=http://127.0.0.1:10808 -c https.proxy=http://127.0.0.1:10808 <命令>`
     - **环境变量方式（适用于 curl/wget/pip/npm 等）**：在命令前设置 `http_proxy=http://127.0.0.1:10808 https_proxy=http://127.0.0.1:10808`
   - 如果代理也失败，再向我报告具体错误信息。
